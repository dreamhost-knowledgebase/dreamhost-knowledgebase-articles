Is there a limit to how many domains/subdomains I can add to my panel?

No. There is no limit to how many domains/subdomains you are able to add to your DreamHost account with shared, VPS, or Dedicated Hosting.

Do I have to create a new directory if I add a domain?

No, you do not.

A new directory automatically appears (in the name of your domain) in your user's directory within fifteen minutes of adding the domain. You don't need to create a new directory yourself.

When somebody visits your website, the configuration files for the DreamHost web server tell the visitor to go to the new directory. When you add a new domain and specify its directory, the panel adds the necessary lines to the necessary config files on the web server. There is no need to for any other configuration other than adding the domain in your panel.

How much is it for an extra domain?

Only Shared, VPS, and Dedicated Hosting plans allow an unlimited number of domains and subdomains.

If you want to host an entirely new domain, you can register it on the Manage Registrations page or through any other domain registrar (although, remember that registration costs may vary).

Does an extra domain have to point to the same place as the original domain?

No, but it can if you'd like it to.

It can point to your existing site, or it can point to an entirely different directory. When adding the domain in your panel, you get to decide where it points.

Does an extra domain get its own storage space, login, email aliases, and so on?

Your extra domain gets its own email addresses.

You can also use the same username as your main domain, or you can create a new username. All web files for the domain are stored under the user you assign to it. What if I already have a registration at another company?

If you've already purchased a registration at another company, you can point your DNS to DreamHost. After the DNS is pointing to DreamHost, the domain you've added hosting for will be hosted at DreamHost. View the following article for information on what DNS records you should point to.
