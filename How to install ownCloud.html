There are two ways to install ownCloud on your website.

  • Installing on your website
  • Downloading a client for your local computer

Both options allow you to easily upload data to your DreamObjects account.

Install on your website

Using the web installer

If you encounter problems using the web installer, try the second option to download the files manually and extract it to the desired location on your website.

The following instructions use the web installer method to install ownCloud. It is the easiest installation method since it checks dependencies, sets permissions, and downloads the latest version. You can then complete the installation from your browser.

  1. Right-click here and save the file to your computer. You can also run the following command via SSH to download it to your server: 
    [server]$ wget https://download.owncloud.com/download/community/setup-owncloud.php
  2. If you chose to manually download the file, use the FTP article to upload setup-owncloud.php to your server in the proper domain directory.
  3. Point your web browser to example.com/setup-owncloud.php
  4. Click the Next button, and then follow the instructions to configure ownCloud.
  5. Log into your newly created ownCloud instance.

Installing manually

  1. Visit ownCloud’s download page.
  2. Click 'Download ownCloud Server'.
  3. The next page details all steps to install. Right click either the .tar.bz2 or .zip link, then choose 'Copy link Address'.

    This example uses the .zip file.

  4. Log into your server via SSH.
  5. Navigate to the directory you wish to install ownCloud
  6. Run a 'wget' command to download the .zip file. 
    [server]$ wget https://download.owncloud.org/community/owncloud-10.0.3.zip
  7. Download the .asc file for the zip file. 
    [server]$ wget https://download.owncloud.org/community/owncloud-10.0.3.zip.asc
  8. Next, download the ownCloud gpg .asc file. 
    [server]$ wget https://owncloud.org/owncloud.asc
  9. Import the owncloud.asc file 
    [server]$ gpg --import owncloud.asc
  10. Verify your .zip.asc file against your .zip file. 
    [server]$ gpg --verify owncloud-10.0.3.zip.asc owncloud-10.0.3.zip
gpg: Signature made Fri Sep 15 07:55:36 2017 PDT using RSA key ID F6978A26
gpg: Good signature from "ownCloud <info@owncloud.com>"
  11. If it says 'Good signature', decompress the file: 
    [server]$ unzip owncloud-10.0.3.zip
    This creates a new subdirectory called /owncloud in the same directory you ran the command.
  12. Point your browser to that new /owncloud folder, which most likely appears as example.com/owncloud.
  13. Enter a username and password, and then click the Finish setup button.
