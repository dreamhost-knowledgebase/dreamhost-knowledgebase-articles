If you already have an existing WordPress site, you are able to import its content into your DreamPress site.

Importing to DreamPress as a test environment

The following instructions help you create a safe copy of your DreamPress site for testing purposes, without affecting your existing one. For example, you can set up a new instance of DreamPress on a different domain and then import your existing WordPress site to it. Once you're finished testing, delete it to avoid extra costs.

You can import a current WordPress install into a temporary DreamPress site. This allows you the ability to use the temporary domain as a testing environment before migrating to your actual site URL. View the following articles for details on how to create a DreamPress temporary domain:

  • Creating a DreamPress temporary domain
  • Changing a DreamPress temporary domain to your live site

If you have no need for the testing environment, you can instead upgrade your WordPress install directly to DreamPress. Otherwise, proceed with the steps below to import into an existing DreamPress site.

Importing into DreamPress

  1. Navigate to the (Panel > 'DreamPress' > 'Dashboard') page.
  2. If you want to change the name of your WordPress install to another domain while importing, create a new DreamPress installation.
    If you want to test your import on a temporary domain, create a temporary DreamPress site.
  3. After it's created, click the 'Manage' button.
  4. Click the 'Domain' tab on the bottom left.
  5. On the bottom right, click the titled 'Import existing site'.
    A window pops up allowing you to select a site from which to import your content:
  6. Choose the site you'd like to import data from in the dropdown menu.
  7. Click the Start Import button to begin the import process.

This creates a clone/copy of the WordPress site and changes the domain name. This is ideal for testing DreamPress, testing new plugins or themes, and so on, since it leaves the original site intact. It's also good for splitting out a single WordPress installation from a domain that has multiple apps. For example, you can turn example.com/blog into blog.example.com.

If you're importing your WordPress install from a different domain name, the import toolÂ will attempt to replace the domain/URLs. If it fails to do so, the email following the import will indicate that. In any case, you should always double check if this data is correct after the import has completed.

If you wish to migrate a different domain's WordPress content to a domain that you want on DreamPress, then you should add the domain first to DreamPress and then use the Import feature.
