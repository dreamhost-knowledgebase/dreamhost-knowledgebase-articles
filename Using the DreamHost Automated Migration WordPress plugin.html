Overview

DreamHost has partnered with the makers of BlogVault to provide DreamHost Automated Migration, a free WordPress plugin that allows you to make WordPress site migrations to DreamHost fast and easy.

This plugin works with the vast majority of WordPress sites migrating to DreamHost. If you believe you might have an unusual site configuration or web host, see the FAQs section at the end of the article.

Before you begin a migration

  • Test and install all available updates for WordPress, including all installed themes and plugins. Make sure your site has no errors or hacked code.
  • To avoid any downtime on your site, keep the domain’s DNS pointed at your old host until migration is complete and you’ve had a chance to look at the migrated site at DreamHost.

Installing the DreamHost Automated Migration plugin

  1. If you haven’t already, Add Hosting to your domain in your DreamHost panel. You can add WordPress as a One-Click Install.
    • It’s okay to do this while your domain is hosted at your old host. Adding hosting on DreamHost does not typically affect your old hosting and your existing site will remain live for now.
    • if you are concerned about affecting your existing live site, you can add a temporary subdomain name ending with .dream.press for DreamPress servers, or .dreamhosters.com for all other hosting. This will create extra work at the end of the process if you do this.
  2. Find the following information about your DreamHost account:
    • FTP/SFTP User: Navigate to Manage Domains and make a note of your User to the right of your domain.
    • Host/Server Address: Click the user to find the server name at the top to the right of the username. Example: (on servername) Follow the below server address format for your hosting plan:
      • Shared Hosting plan — servername.dreamhost.com
      • DreamPress or VPS plan — servername.dreamhostps.com
      • Dedicated Server plan — servername.dreamservers.com
    • FTP or SFTP: Double check that your user type is either SFTP or SSH. If it's FTP, switch the user type to SFTP or SSH (which also provides SFTP).
    • FTP/SFTP Password: Set a password for this user and click 'Save Changes'. Have this information ready to enter in the next step.
  3. Install and activate the DreamHost Automated Migration plugin on the WordPress site at your old host:
    1. From the WordPress Admin Dashboard for your site, go to Plugins > Add New.
    2. In the search field, type 'DreamHost Automated Migration' and click the Install button for the correct plugin.
    3. Click Activate. The Migration page opens automatically.
    4. Enter an email address where you would like to receive notifications about migration status and completion.
    5. Enter the web address (Destination Site URL) for your site. Usually this is the same address as your site on your current host. If you are using a DreamHost temporary subdomain or if you will be changing your domain for your site, use that address instead. Make sure to enter 'https://' at the beginning of the web address if you plan to use a secure certificate.
    6. Enter the FTP information you obtained in Step 2.
    7. Review and agree to the Terms of Service and Privacy Policy if you agree and wish to continue.
    8. Click the Migrate button.
  4. Bookmark the URL to check your migration’s progress and you close the browser tab or window while it is running. Most migrations take under an hour, but if your site is larger than a couple GB of data, it can take more than an hour or two. Once completed, you will receive a notification email at the address you entered during the step above.

Checking your migrated site and making it live on DreamHost

If your domain is the same on DreamHost and the old host

Add '.dream.website' to the end of your domain to view the migrated site on DreamHost. For example, if your domain is 'example.com', view your migrated site at 'example.com.dream.website'.

Once you are satisfied with the migrated site, change your domain’s DNS to point to DreamHost.

If you chose a different domain name on DreamHost from the old host

You can use the different domain name to view your site live on DreamHost. Make sure any new domain name you chose is already using DreamHost DNS records.

If you are using a temporary sub-domain name ending in .dreamhosters.com or .dream.press

You can use the temporary sub-domain name immediately to view the migrated site.

  • On DreamPress, when you are ready to make your site live on DreamHost using your real domain, follow these instructions to rename your domain.
  • On all other hosting plans, when you are ready to make your site live on DreamHost using your real domain name, follow these instructions to rename your domain.

FAQs

Can I use this plugin to migrate a WP multisite?

Yes, the Migrate to DreamHost plugin allows the migration of a WordPress network or Multisite install. After installing the plugin, make sure to 'Network Activate' it to ensure it can run from the Network Admin dashboard. However, please note the following considerations:

  • The migration must move the whole network to DreamHost. It does not allow the migration of individual subsites, or the import of a subsite into a different network.
  • The recommended hosting plans for WordPress Multisite is a DreamHost VPS, Dedicated, or DreamCompute. Multisite installs on Shared and DreamPress plans are not recommended or supported.
  • DreamHost Support does not offer technical assistance with Multisite installs.

Are there any restrictions or limitations?

The plugin will work with the vast majority of WordPress sites, themes, plugins and configurations.

However, please note that WordPress sites on WordPress.com cannot be migrated using this or any other plugin as an exact copy of the original site to any other host. You can however import the content of your pages, posts, and embedded images using the instructions found on this page.

Your site must already be a WordPress site in order to migrate an exact copy of your site using this or any other plugin. This means that the Migrate to DreamHost plugin cannot be used to migrate other types of sites, such as those built using Wix, Weebly, SquareSpace, Joomla, or other site builders.

At this time, you can migrate a WordPress site of any size using this plugin. Please note the time to complete the migration increases for larger sites.
