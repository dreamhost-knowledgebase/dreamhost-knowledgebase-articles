In an effort to improve email services, DreamHost has created a new mail server and is currently migrating existing customers to it. This will ultimately help to stabilize and improve email services for all customers.

SSL connections are also enabled by default ensuring your connecting to the server is safe and secure.

Please note that due to the email server upgrades, Vanity mail servers will no longer be supported. View the section below for further details.

What to expect

Duplicate emails during the migration process

This could potentially happen if any email is set as 'READ'. It could also happen to emails that were read, but then marked as 'UNREAD'.

DNS propagation issues

It's possible that you may notice some email delays during the migration period. It's possibly this could result in missing email as some email could potentially be routed to the old server during the DNS propagation period. However, it will eventually be copied to the new server.

How to prepare

Updating your email client configuration

If you have been migrated to the mail server, your Email client configuration may need to be updated.

New mail servers

In the past, your email servername (hostname) would have looked like 'sub4.mail.dreamhost.com'. This address will still continue to work and these easier-to-remember names will also work:

IMAP (most common)

  • imap.dreamhost.com — Port 993 (incoming IMAP connections)
  • smtp.dreamhost.com — Port 465 (outgoing SMTP connections)

POP

  • pop.dreamhost.com — Port 995 (incoming POP connections)
  • smtp.dreamhost.com — Port 465 (outgoing SMTP connections)

Webmail server

You can log into your webmail at the following address:

  • webmail.dreamhost.com

What if I'm using external DNS?

If you are using an external DNS provider, DNS may need to update (including MX records).

MX servers

The following are the new MX servers:

  • mx1.dreamhost.com
  • mx2.dreamhost.com
  • mx3.dreamhost.com

Vanity Mail Server EOL

After your email service is migrated to the new server, DreamHost will no longer support the use of mail.yourdomain.com or webmail.yourdomain.com. webmail.yourdomain.com will continue to work, but eventually it will stop.

It's highly recommended that you only use webmail.dreamhost.com going forward. This will help to fix the issue of receiving a 'mismatch' error in your browser when visiting:

  • Certificate domain mismatch error when connecting to a DreamHost mail server
