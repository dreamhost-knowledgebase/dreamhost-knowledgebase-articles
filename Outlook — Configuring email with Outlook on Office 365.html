Overview

The following describes how to set up and configure your DreamHost email address in Outlook on Office 365.

To quickly find your email settings in the panel, open the Manage Email page and look for the link in the upper right. Click the text to open the quick-access instructions for your email address:

Incoming
  • imap.dreamhost.com
  • pop.dreamhost.com
Outgoing
  • smtp.dreamhost.com

Setting up Outlook

To add an email account to Outlook on Office 365 in Windows:

  1. Open Outlook on Office 365 from your start menu.
  2. On the top left, click the File tab.
    A popout menu appears prompting you to add an account:
  3. Click + Add Account.
  4. Enter your email address.
  5. Click the Advanced options link and then check the box to set up the account manually.
  6. Click the Connect button.
  7. Select either POP or IMAP. IMAP is recommended.
  8. On the IMAP Account Settings page, enter your Incoming and Outgoing mail settings:
    • Server — This is your DreamHost mail server name. The example above uses imap.dreamhost.com for the incoming server. If you'd prefer a POP connection, you can use pop.dreamhost.com instead. The outgoing server is always smtp.dreamhost.com.

      Make sure you only enter your DreamHost mail servername. Do not use a subdomain such as mail.example.com. You may get an insecure warning if you use the wrong mail hostname.

    • Port — If you chose IMAP, the secure port is 993. For outgoing ports, use 465 or 587.

      Port 465 with SSL is recommended, however, some email clients are unable to use this port.

      If you are unable to use port 465, the next best option is port 587 using STARTTLS.

    • Encryption — Select SSL/TLS from the drop-down menu.
    • Require logon using Secure Password Authentication — Check this box.
  9. Click the Next button.
  10. Enter your password and click Connect.

Forwarding errors

You may see the following error when using the forward feature within Outlook:

Remote Server returned '550 5.7.520 Access denied, Your organization does not allow
external forwarding. Please contact your administrator for further assistance. AS(7555)'

Instructions for resolving this error can be found in the following Microsoft documentation:
