Using Cloudflare with WWW or a subdomain

If you're using Cloudflare without a subdomain, then you must select ‘Add WWW’ to the domain. This is a limitation that Cloudflare imposes when adding their service through DreamHost. If you wish to use Cloudflare without the www subdomain, you must purchase an account directly through their website.

If your domain is already a subdomain, such as subdomain.example.com, then there is no need to change the domain hosting options and you can configure Cloudflare on subdomain.example.com.

If your domain uses a content management application like WordPress, this can cause an issue as the URL used by the application has changed. If you need to change your domain to include WWW in the URL, you must make sure to update the URL used in your application or code as well. It’s best to make the changes to your CMS program or code before enabling Cloudflare on the domain.

View the Change WordPress Site URL article for details on how to adjust this setting for a WordPress site.

Using Cloudflare with dreamhosters.com subdomains

Cloudflare cannot be used with dreamhosters.com subdomains at this time.

Using Cloudflare with external nameservers

DreamHost nameservers

If your domain's nameservers are pointed to DreamHost, the Cloudflare plan you added to your DreamHost panel will automatically connect to your domain.

NON-DreamHost nameservers

If your domain's nameservers are NOT pointed to DreamHost, its DNS records are controlled at a different company. In order to use the Cloudflare subscription in your DreamHost panel, you must point your DNS to DreamHost using one of the following options:

  • Point your nameservers (recommended)
  • Point individual A Records

The best option is to point your nameservers to DreamHost since this points ALL of your DNS records to DreamHost.

The only reason to not point your nameservers is if you have specific services you need to continue hosting at another company. In that case, you would need to manually set up individual A records at the host where your domain's nameservers are currently managed. View the following article for instructions on how to confirm where your domain's nameservers are pointed.

  • Checking where your Nameservers are pointed

You can then use the following article to locate your DreamHost domain's IP addresses. These are the values you use to create A Records at the company where your nameservers are pointing.

  • What DreamHost DNS records do I point my site to?

You'll need to create the following A Records.

  • example.com
  • www.example.com

Or if you're using a subdomain:

  • subdomain.example.com

The new DNS records will then take a few hours to update online.

Changing the Cloudflare account

It's only possible to change the account within the primary domain (www.example.com). Subdomains such as blog.example.com cannot change the Cloudflare account assigned to them. View the following article for further information:
